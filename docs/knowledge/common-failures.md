# 常见网络故障知识库

> 这是 NetPilot 知识库的种子文件(未来用于 RAG 检索增强)。每条按
> **现象 / 根因 / 如何确认 / 处置** 组织,可直接喂给 LLM 增强判断。

---

## 1. ICMP 被过滤("ping 不通但服务正常")

- **现象**:ping 目标 100% 丢包,但网页/接口实际可用。
- **根因**:云主机/防火墙禁用 ICMP,链路与 TCP 服务其实正常。
- **如何确认**:`icmp_ping` 失败后用 `tcp_ping` 复核——TCP 端口可达即证实。
- **处置**:无需处理网络;若监控依赖 ICMP,改用 TCP 探活。

## 2. DNS 解析到陈旧/错误 IP

- **现象**:域名能解析,但连到的是错误服务器(旧节点/已下线主机)。
- **根因**:DNS 缓存未更新、TTL 过长、权威记录改错、DNS 污染。
- **如何确认**:`dns_lookup` 看解析到的 IP 是否符合预期;对比权威 DNS 与本地 DNS 结果。
- **处置**:刷新本机/递归 DNS 缓存;核对权威记录与 TTL。

## 3. MTU 黑洞

- **现象**:小包通、大包丢;HTTPS 握手卡死、大文件传输出错。
- **根因**:链路某段 MTU 小于默认 1500 且 ICMP "fragmentation needed" 被丢(PMTUD 失败)。
- **如何确认**:大包 ping 丢包而小包不丢;traceroute 无明显断点却时通时断。
- **处置**:链路上启用 PMTUD;或在主机/隧道侧降低 MTU(如 VPN 隧道用 1400)。

## 4. ARP 冲突("时通时不通")

- **现象**:同一网段内目标偶发性不可达,MAC 地址漂移。
- **根因**:两台设备配了相同 IP,ARP 表在两者间反复刷新。
- **如何确认**:arp 表中同一 IP 对应多个 MAC;丢包呈周期性。
- **处置**:定位重复 IP 的设备并修正配置。

## 5. TIME_WAIT / 连接数耗尽

- **现象**:突发大量短连接后服务"假死",新连接被拒。
- **根因**:TCP TIME_WAIT 堆积或端口/句柄耗尽。
- **如何确认**:`tcp_ping` 偶发失败;服务端 `ss`/`netstat` 见大量 TIME_WAIT。
- **处置**:启用连接复用(keepalive/连接池);调内核 `tcp_tw_reuse`、扩大端口范围。

## 6. 证书过期 / 链不全 / 域名不匹配

- **现象**:HTTPS 报 `NET::ERR_CERT_*`;部分客户端(老 Java/curl)无法验证。
- **根因**:证书到期未续、缺中间证书、SNI 配置错、用 IP 访问 HTTPS。
- **如何确认**:`tls_inspect` 看 `days_left`/`verify_error`/`hostname_matches`。
- **处置**:续签并部署完整证书链;确保 SNI 与访问域名一致。

## 7. 防火墙 / 安全组过滤端口

- **现象**:服务已启动但端口外部不可达。
- **根因**:云安全组、主机防火墙(iptables/Windows 防火墙)未放行。
- **如何确认**:`port_scan` 显示目标端口为 `filtered`(超时)而非 `closed`(RST)。
- **处置**:在安全组/防火墙放行对应端口与来源。

## 8. HTTP 5xx(应用侧故障)

- **现象**:网络各层正常,但 HTTP 返回 500/502/503。
- **根因**:应用内部错误、上游不可用、限流熔断。
- **如何确认**:`http_probe` 拿到 5xx,而 DNS/连通性/TLS 全 OK。
- **处置**:**非网络问题**,工单转应用/后端团队。

## 9. 本机代理/hosts 干扰

- **现象**:仅本机访问异常,他人正常。
- **根因**:本机配置了失效代理、hosts 静态映射到错误 IP。
- **如何确认**:`local_check` 发现代理或 hosts 覆盖。
- **处置**:修正代理或删除/更新 hosts 条目。

## 10. 变更导致的中断(强信号)

- **现象**:刚变更/上线后立即故障。
- **根因**:DNS 记录改错、防火墙规则、路由变更、证书未续。
- **如何确认**:核对近 24h 变更记录,优先复验被改对象。
- **处置**:回滚或修正变更项。
