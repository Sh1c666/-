# 网络排查分诊决策树

这是 NetPilot Agent 内化的排查"算法",也是运维老手脑子里的假设—验证循环。
来源:真实运维高频场景。每一层用一个工具验证,根据结果决定下一步,**不跳步、不瞎猜**。

## 主场景:服务访问异常

```
症状:访问某服务/网页异常
│
├─ ① 本机自检  local_check
│   ├─ 代理/hosts/DNS 配置异常 → 本机问题(常被误当网络故障)
│   └─ 正常 → 进入 ②
│
├─ ② DNS 解析  dns_lookup
│   ├─ NXDOMAIN / 超时       → DNS 层问题(本机 DNS?域名错?权威挂?)
│   ├─ 解析到陈旧/错误 IP     → DNS 缓存 / 配置错 / 污染
│   └─ 正常 → 进入 ③
│
├─ ③ 连通性  icmp_ping ──► (失败时) tcp_ping 复核
│   ├─ ICMP 不通但 TCP 通    → 目标禁 ICMP,链路其实 OK ★经典坑
│   ├─ 全不通                → traceroute 定位断点
│   ├─ 丢包 / 高延迟         → 链路质量差,traceroute 找拥塞跳
│   └─ 正常 → 进入 ④
│
├─ ④ 路径定位  traceroute(仅当 ③ 异常时)
│   ├─ 到达目标              → 三层可达,断点不在路径
│   ├─ 第 N 跳中断           → 问题在该跳前后链路(机房?运营商?)
│   └─ 部分超时              → 中间节点限速/过滤,结合丢包判断
│
├─ ⑤ 端口可达性  port_scan / tcp_ping
│   ├─ closed(收到 RST)     → 主机可达,服务没起 → 应用/服务问题
│   ├─ filtered(超时)       → 防火墙 / 云安全组丢包
│   └─ open → 进入 ⑥
│
├─ ⑥ TLS 握手  tls_inspect(仅 HTTPS / 443)
│   ├─ 过期 / 剩余 ≤30 天    → 证书问题(给剩余天数)
│   ├─ 域名不匹配             → SNI 配置错 / 用 IP 访问 HTTPS
│   ├─ 自签名 / 链不全        → 浏览器拦截
│   └─ 正常 → 进入 ⑦
│
└─ ⑦ HTTP 请求  http_probe
    ├─ 5xx                   → 服务端问题 → 锅还给应用,排除网络 ★
    ├─ 4xx                   → 客户端 / 权限 / 路由
    ├─ 慢但 200              → 对比 TCP RTT:TCP 快 HTTP 慢 → 应用慢
    └─ 一切正常              → 八成是客户端本机(代理/hosts/缓存/本地防火墙)
```

## 高频反直觉陷阱(必须内化)

1. **ping 不通 ≠ 网络不通。** 云主机/防火墙常禁 ICMP,但 TCP 端口通。看到丢包先 `tcp_ping` 复核。
2. **DNS 解析到 IP ≠ 解析正确。** 可能是陈旧/死节点/被污染的 IP。
3. **"慢"常常不是带宽,是丢包与重传。** 1% 丢包即可让 TCP 吞吐崩塌。
4. **closed vs filtered:** closed(收到 RST)= 服务没起;filtered(超时)= 防火墙丢包。
5. **HTTP 5xx = 应用侧。** 请求已到后端,不是网络,工单转应用团队。
6. **变更信号:** 80% 重大故障前 24 小时有变更。故障若发生在上线后,优先怀疑 DNS 改动 / 防火墙规则 / 证书未续 / 路由变更。
7. **本机问题伪装成网络问题:** 本机防火墙、hosts、代理、本机负载爆表。排查网络前先 `local_check`。

## 按现象快速归类

| 现象 | 首要怀疑层 |
|---|---|
| 完全打不开 | DNS → 防火墙/安全组 → 服务未启动 → 证书 |
| 卡 / 慢 | 链路丢包/延迟 → 带宽 → MTU 黑洞 → 后端慢 |
| 时通时不通 | ARP 冲突 → DNS 轮询到死节点 → TIME_WAIT 耗尽 → 路由抖动 |
| 上线后挂 | DNS 改动 → 防火墙规则 → 路由变更 → 证书未续 |
| HTTPS 报错 | 证书过期/链不全 → TLS 版本 → SNI |
